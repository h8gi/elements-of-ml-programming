structure <NewStructureNAme> = <functorName>(<structureArgument>)

1.1.1. 8.3.3 More General Forms for Functor Parameters and Arguments

functorの引数にはstructure名でなくinlineのstructを書くことも可能。 structure を明示する書き方もある。 

functor <identifier>(structure S1:Sig1 and ... and Sn:Sign)
        = <structure definition>

functorの引数は、signature内で宣言可能なものと同一である。(typeやval、exceptionなど) parameterを渡すときにはspecification keyword(structure, type, valなど)を引数名のようにして使う。 

structure Bar = Foo(structure I = Int; structure R = Real)
structure Bar = Foo(structure I = Int and R = Real)
structure Bar = Foo(structure I = int; structure R = Real; val x = 2)
structure Bar = Foo(structure I = int and R = Real; val x = 2)

1.2. 8.4 Sharings

読む前の理解: 異なる場所に出現しているが実際は同じであるはずのものを、これらは同じであると宣言する機能

1.2.1. 8.4.1 Sharing Specifications

sharing specificationはsignatureの中に現れる。 

sharing type <type> = <type> = ... = <type>
sharing <structure> = <sturecutre> = ... = <structure>

type sharingが基本。 structure sharingは複数のstructure内のidenticalなtype全部をsharingする略記法。 structure同士がequalであるという意味ではない。

1.2.2. 8.4.2 Substructures

signature内でstructureを宣言しておける。その場合substructureのsigは必須。 structure内でstructureを定義することもできる。その場合sigはなくても良い。(定義や既存のstructureから導出できるので)
